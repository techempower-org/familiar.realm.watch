# familiar v0.2 — "The familiar remembers better"

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Harden familiar with local llama.cpp inference on katana, multi-endpoint failover with circuit breakers, Emmimal retrieval components 2–4 (rerank, decay, extractive compression), a confidence gate + stuck detector, `/api/familiar/eval`, an MCP server, diary buffering, and PWA citation popovers.

**Architecture:** familiar-api gains a `LlamaCppClient` (OpenAI-compat HTTP to katana's `llama-server`) and an `InferenceRouter` that tries endpoints in priority order — katana llama.cpp first, familiar's Ollama second — wrapping each in the existing `CircuitBreaker`. The retrieval pipeline in `memory-protocol.ts` gains three post-search steps: domain-weighted rerank, exponential temporal decay, and extractive compression (Emmimal components 2, 3, 4). A diary buffer accumulates post-stream writes and flushes every 10 turns. The MCP server exposes `familiar_chat`, `familiar_recall`, and `familiar_reflect` via `@modelcontextprotocol/sdk` over HTTP. The PWA replaces `[drawer_xxx]` citation markers with hover popovers built entirely from DOM methods.

**Tech Stack:** TypeScript + Bun, `@modelcontextprotocol/sdk` (^1.29, already installed), llama.cpp (cmake build with CUDA on katana), `bun:test` for all unit tests.

**Key constraints:**
- Bun, not Node — `bun:test`, `Bun.spawnSync({ cmd: [...] })`, native TS.
- No `innerHTML` anywhere in web JS — use `document.createElement` + `textContent`.
- No subprocess shell strings — use argv arrays with `Bun.spawnSync`.
- Emmimal retrieval components are client-side TypeScript in `src/retrieval/`, applied to results returned by palace-daemon.

**Why this is a prerequisite:** memorypalace fork (270 ahead of upstream) introduced Stop-hook checkpoints (~5-word session summaries like `"CHECKPOINT: fixed auth, started dashboard"`). They have high token density and dominate vector similarity if not filtered. The fork added `kind=` parameter — `"content"` (default, excludes checkpoints), `"checkpoint"` (audit/recovery), `"all"` (legacy). Without `kind="content"`, every familiar retrieval is polluted by checkpoint noise on heavily-autobiographical palaces (>50% checkpoints by count). This is a v0.1-era gap that v0.2 must close before anything else.

Validated on 151K-drawer palace (palace-daemon CHANGELOG, v1.5.1): same query returned 5 substantive drawers with `kind=content` vs. 5 CHECKPOINTs with old behavior. Daemon `/search` and `/context` accept the param as of v1.5.1.

```typescript
import type { ChatStreamOpts } from "./ollama-client.ts";
import type { OllamaChatChunk } from "./types.ts";

export interface LlamaCppClientOptions {
  baseUrl: string;
  model: string;
  fetch?: typeof fetch;
}

/**
 * HTTP client for llama.cpp's OpenAI-compatible server (llama-server).
 * Emits OllamaChatChunk-shaped objects so it can substitute for OllamaClient
 * in InferenceRouter without changes to chat.ts.
 */
export class LlamaCppClient {
  private baseUrl: string;
  private model: string;
  private fetchFn: typeof fetch;

  constructor(opts: LlamaCppClientOptions) {
    this.baseUrl = opts.baseUrl.replace(/\/$/, "");
    this.model = opts.model;
    this.fetchFn = opts.fetch ?? fetch;
  }

  async isHealthy(): Promise<boolean> {
    try {
      const res = await this.fetchFn(`${this.baseUrl}/health`);
      return res.ok;
    } catch {
      return false;
    }
  }

  async *chatStream(opts: ChatStreamOpts): AsyncGenerator<OllamaChatChunk> {
    const body = {
      model: opts.model ?? this.model,
      messages: opts.messages,
      stream: true,
      options: opts.options,
    };
    const res = await this.fetchFn(`${this.baseUrl}/v1/chat/completions`, {
      method: "POST",
      headers: { "content-type": "application/json" },
      body: JSON.stringify(body),
    });
    if (!res.ok) throw new Error(`llama-cpp chat: ${res.status} ${res.statusText}`);
    if (!res.body) throw new Error("llama-cpp chat: no response body");

    const reader = res.body.getReader();
    const decoder = new TextDecoder();
    let buffer = "";

    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      buffer += decoder.decode(value, { stream: true });

      let newlineIdx: number;
      while ((newlineIdx = buffer.indexOf("\n")) >= 0) {
        const line = buffer.slice(0, newlineIdx).trim();
        buffer = buffer.slice(newlineIdx + 1);
        if (!line.startsWith("data:")) continue;
        const payload = line.slice(5).trim();
        if (payload === "[DONE]") {
          yield { model: this.model, created_at: new Date().toISOString(), done: true };
          return;
        }
        try {
          const parsed = JSON.parse(payload);
          const delta = parsed.choices?.[0]?.delta?.content ?? "";
          const isDone = parsed.choices?.[0]?.finish_reason === "stop";
          yield {
            model: this.model,
            created_at: new Date().toISOString(),
            message: delta ? { role: "assistant", content: delta } : undefined,
            done: isDone,
          } as OllamaChatChunk;
        } catch { /* skip malformed SSE lines */ }
      }
    }
    yield { model: this.model, created_at: new Date().toISOString(), done: true };
  }
}
```

```typescript
import { CircuitBreaker, type CircuitBreakerOptions } from "./circuit-breaker.ts";
import type { ChatStreamOpts } from "./ollama-client.ts";
import type { InferenceChatProvider, OllamaChatChunk } from "./types.ts";

const DEFAULT_BREAKER: CircuitBreakerOptions = {
  threshold: 3,
  windowMs: 30_000,
  openMs: 60_000,
};

export class InferenceRouter implements InferenceChatProvider {
  private providers: InferenceChatProvider[];
  private breakers: CircuitBreaker[];

  constructor(providers: InferenceChatProvider[], breakerOpts?: Partial<CircuitBreakerOptions>) {
    this.providers = providers;
    const opts = { ...DEFAULT_BREAKER, ...breakerOpts };
    this.breakers = providers.map(() => new CircuitBreaker(opts));
  }

  /** Expose breaker by index for testing. */
  breakerFor(i: number): CircuitBreaker {
    return this.breakers[i];
  }

  async isHealthy(): Promise<boolean> {
    for (const p of this.providers) {
      if (await p.isHealthy()) return true;
    }
    return false;
  }

  async *chatStream(opts: ChatStreamOpts): AsyncGenerator<OllamaChatChunk> {
    for (let i = 0; i < this.providers.length; i++) {
      const breaker = this.breakers[i];
      if (breaker.state() === "open") continue;
      try {
        yield* await this._streamWithBreaker(this.providers[i], breaker, opts);
        return;
      } catch {
        // breaker already recorded the failure; try next
      }
    }
    throw new Error("all inference endpoints failed or circuit-broken");
  }

  private async _streamWithBreaker(
    provider: InferenceChatProvider,
    breaker: CircuitBreaker,
    opts: ChatStreamOpts
  ): Promise<AsyncGenerator<OllamaChatChunk>> {
    // We need to wrap the generator inside breaker.run() so failures record.
    // Strategy: eagerly pull the first value inside breaker.run; if it throws,
    // the breaker records it. Then yield buffered + remainder outside.
    const gen = provider.chatStream(opts);
    const buffer: OllamaChatChunk[] = [];

    await breaker.run(async () => {
      const first = await gen.next();
      if (!first.done) buffer.push(first.value);
    });

    async function* emit(): AsyncGenerator<OllamaChatChunk> {
      yield* buffer;
      for await (const chunk of gen) yield chunk;
    }
    return emit();
  }
}
```

### Note on PalaceDrawer field handling

After Task 0.2, `PalaceDrawer` carries `topic`, `matched_via`, `cosine`, `bm25` from the memorypalace fork. Treat `cosine` and `bm25` as **read-only raw scores** — never overwrite them. The pattern below — `{ ...d, similarity: newScore }` — preserves all new fields automatically. No special handling needed in rerank/decay/compress; just don't accidentally drop the spread.

```typescript
import type { PalaceDrawer } from "../types.ts";

const WING_WEIGHT = 0.68;
const TAG_WEIGHT = 0.32;
const WING_BOOST = 1.4;
const RECENCY_BONUS = 0.1;
const RECENCY_WINDOW_MS = 48 * 3600 * 1000;

function tagImportance(drawer: PalaceDrawer, wingScope: string | null): number {
  if (wingScope && drawer.wing === wingScope) return WING_BOOST;
  return 1.0;
}

function recencyBonus(drawer: PalaceDrawer): number {
  if (!drawer.created_at) return 0;
  try {
    const age = Date.now() - new Date(drawer.created_at).getTime();
    return age <= RECENCY_WINDOW_MS ? RECENCY_BONUS : 0;
  } catch {
    return 0;
  }
}

function finalScore(drawer: PalaceDrawer, wingScope: string | null): number {
  const base = drawer.similarity ?? drawer.distance ?? 0;
  return base * WING_WEIGHT + tagImportance(drawer, wingScope) * TAG_WEIGHT + recencyBonus(drawer);
}

/**
 * Rerank drawers by domain-weighted score.
 * Returns a new array sorted descending; mutates similarity field with final score.
 */
export function domainRerank(drawers: PalaceDrawer[], wingScope: string | null): PalaceDrawer[] {
  return drawers
    .map((d) => ({ ...d, similarity: finalScore(d, wingScope) }))
    .sort((a, b) => (b.similarity ?? 0) - (a.similarity ?? 0));
}
```

```typescript
import type { PalaceDrawer } from "../types.ts";

export interface DecayOptions {
  halfLifeDays: number;
}

export function temporalDecay(drawers: PalaceDrawer[], opts: DecayOptions): PalaceDrawer[] {
  const lambda = Math.log(2) / opts.halfLifeDays;
  const now = Date.now();

  return drawers.map((d) => {
    const base = d.similarity ?? d.distance ?? 0;
    if (!d.created_at) return { ...d, similarity: base };
    try {
      const ageDays = (now - new Date(d.created_at).getTime()) / 86400_000;
      const decayed = base * Math.exp(-lambda * ageDays);
      return { ...d, similarity: decayed };
    } catch {
      return { ...d, similarity: base };
    }
  });
}
```

```typescript
import type { PalaceDrawer } from "../types.ts";

const SHORT_THRESHOLD = 500;
const MAX_SENTENCES = 3;

function tokenOverlap(a: string, b: string): number {
  const aWords = new Set(a.toLowerCase().match(/\b\w+\b/g) ?? []);
  const bWords = new Set(b.toLowerCase().match(/\b\w+\b/g) ?? []);
  let hits = 0;
  for (const w of aWords) if (bWords.has(w)) hits++;
  return hits / Math.max(aWords.size, 1);
}

function splitSentences(text: string): string[] {
  return text.split(/(?<=[.!?])\s+/).filter((s) => s.trim().length > 0);
}

function compressDrawer(drawer: PalaceDrawer, query: string): PalaceDrawer {
  if (drawer.text.length <= SHORT_THRESHOLD) return drawer;

  const sentences = splitSentences(drawer.text);
  if (sentences.length <= MAX_SENTENCES) return drawer;

  const scored = sentences.map((s, idx) => ({ s, score: tokenOverlap(s, query), idx }));
  const top = scored
    .sort((a, b) => b.score - a.score)
    .slice(0, MAX_SENTENCES)
    .sort((a, b) => a.idx - b.idx) // restore original order
    .map((x) => x.s);

  return { ...drawer, text: top.join(" ") };
}

export function extractiveCompress(drawers: PalaceDrawer[], query: string): PalaceDrawer[] {
  return drawers.map((d) => compressDrawer(d, query));
}
```

## Phase 5: Eval endpoint — SME adapter contract

### Why this phase changed shape

The earlier draft of Phase 5 invented a keyword-scoring eval endpoint. The right integration is **multipass-structural-memory-eval's `SMEAdapter` contract** (see `~/Projects/multipass-structural-memory-eval/docs/sme_spec_v8.md`, Category 9 "Handshake"). multipass already has three adapters (`flat_baseline`, `mempalace`, `mempalace-daemon`); familiar becomes the fourth. The scoring lives in multipass; familiar just exposes its retrieval pipeline as a callable surface.

The contract — direct from the spec:

> Your endpoint receives `POST /api/familiar/eval` with `{ query: string, limit?: number }` and returns `{ answer, context_string, retrieved_entities, retrieved_edges, error }`. The `context_string` is the **exact text** that would go into the LLM. SME tokenizes it with tiktoken (cl100k_base) to score tokens-per-correct-answer. Your endpoint cannot game the count by returning a differently-formatted response.

This means familiar's eval endpoint is essentially: "run a turn end-to-end, return what you saw + what you said + the verbatim text you sent the model." Cheap, honest, contract-aligned.

```typescript
import type { PalaceClient } from "../palace-client.ts";
import type {
  Config,
  InferenceChatProvider,
  SmeEntity,
  SmeQueryRequest,
  SmeQueryResponse,
} from "../types.ts";
import { retrieveAndGround } from "../memory-protocol.ts";

export interface EvalRouteDeps {
  cfg: Config;
  palace: PalaceClient;
  inference: InferenceChatProvider;
}

const STUB_ANSWER = "(mock=true: inference skipped — context_string is the only meaningful field)";

export async function handleEval(req: Request, deps: EvalRouteDeps): Promise<Response> {
  let body: SmeQueryRequest;
  try {
    body = (await req.json()) as SmeQueryRequest;
  } catch {
    return new Response(JSON.stringify({ error: "invalid JSON body" }), {
      status: 400, headers: { "content-type": "application/json" },
    });
  }

  if (!body.query || typeof body.query !== "string") {
    return new Response(JSON.stringify({ error: "query required" }), {
      status: 400, headers: { "content-type": "application/json" },
    });
  }

  const limit = body.limit ?? deps.cfg.retrievalLimit;
  const kind = body.kind ?? "content";
  const warnings: string[] = [];
  let availableInScope: number | undefined;
  let retrievedEntities: SmeEntity[] = [];
  let contextString = "";

  // Run retrieval + grounding (the same path the chat route uses)
  let grounded: Awaited<ReturnType<typeof retrieveAndGround>>;
  try {
    grounded = await retrieveAndGround({
      palace: deps.palace,
      userMessage: body.query,
      wingScope: body.wing ?? null,
      retrievalLimit: limit,
      contextBudgetTokens: deps.cfg.tokenBudget.context,
      recentCitations: [],
      kind,
    });
    contextString = grounded.systemPrompt;
    availableInScope = grounded.availableInScope;
    retrievedEntities = grounded.entities;  // memory-protocol.ts must surface these
    warnings.push(...grounded.warnings);
  } catch {
    warnings.push("palace_unreachable");
    contextString = "(palace unreachable — no context)";
  }

  // Generate (unless mock=true)
  let answer = STUB_ANSWER;
  if (!body.mock) {
    try {
      const messages = [
        { role: "system" as const, content: contextString },
        { role: "user" as const, content: body.query },
      ];
      let acc = "";
      for await (const chunk of deps.inference.chatStream({ messages })) {
        acc += chunk.message?.content ?? "";
        if (chunk.done) break;
      }
      answer = acc.trim() || "(empty response from inference)";
    } catch (err) {
      warnings.push("inference_failed");
      answer = `(inference failed: ${err})`;
    }
  }

  const response: SmeQueryResponse = {
    answer,
    context_string: contextString,
    retrieved_entities: retrievedEntities,
    retrieved_edges: [],  // v0.2: empty. v0.3+: KG triples for these entities.
    error: null,
    warnings,
    ...(availableInScope !== undefined ? { available_in_scope: availableInScope } : {}),
  };

  return new Response(JSON.stringify(response, null, 2), {
    status: 200,
    headers: { "content-type": "application/json" },
  });
}
```

## Phase 5b: Trace abstraction + `/api/familiar/graph` proxy

### Why this phase exists

Two separate-but-related observations from the daemon and multipass docs converged on the same idea:

1. **Multipass's `/api/familiar/eval` already requires `context_string` + `retrieved_entities`** — the verbatim system prompt and the entities surfaced. That is a *trace* of one turn.
2. **palace-daemon v1.6.0 added `GET /graph`** — a single-shot structural snapshot (~0.4s vs. 30s+ MCP fan-out on a 151K-drawer palace). multipass already consumes it.

Combining them: every chat turn already produces a structured story (query, drawers walked, system prompt sent, answer streamed, citations cited). Naming this story — `Trace` — costs ~30 lines of TS and pays for itself across at least four downstream features:

- The eval endpoint (already returns the same shape)
- The PWA citation popovers (Phase 8 — popover content *is* the trace)
- mempalace-viz integration (visualize a trace as a graph walk)
- hermes-agent fine-tune corpus (every trace is one training tuple)
- multipass Cat 9 (HandshakeTrace is the same concept, one level up)

This phase just gives the data a name and exposes the daemon's `/graph` endpoint via familiar-api. No new logic; the trace is *already happening*.

```typescript
import type { PalaceClient } from "../palace-client.ts";

export interface GraphRouteDeps {
  palace: PalaceClient;
}

let cache: { ts: number; data: unknown } | null = null;
const TTL_MS = 30_000;  // /graph is heavy on big palaces; cache 30s

export async function handleGraph(_req: Request, deps: GraphRouteDeps): Promise<Response> {
  const now = Date.now();
  if (cache && now - cache.ts < TTL_MS) {
    return new Response(JSON.stringify(cache.data), {
      status: 200,
      headers: { "content-type": "application/json", "x-graph-cache": "hit" },
    });
  }
  try {
    const data = await deps.palace.getGraph();
    cache = { ts: now, data };
    return new Response(JSON.stringify(data), {
      status: 200,
      headers: { "content-type": "application/json", "x-graph-cache": "miss" },
    });
  } catch (err) {
    return new Response(JSON.stringify({ error: String(err) }), {
      status: 502,
      headers: { "content-type": "application/json" },
    });
  }
}
```

Create `src/diary-buffer.ts`:

```typescript
type FlushFn = (entries: string[]) => Promise<void>;

export interface DiaryBufferOptions {
  flushSize: number;
  flushFn: FlushFn;
}

export class DiaryBuffer {
  private entries: string[] = [];
  private flushFn: FlushFn;
  private flushSize: number;
  private flushing = false;

  constructor(opts: DiaryBufferOptions) {
    this.flushSize = opts.flushSize;
    this.flushFn = opts.flushFn;
  }

  size(): number { return this.entries.length; }

  async add(entry: string): Promise<void> {
    this.entries.push(entry);
    if (this.entries.length >= this.flushSize) {
      await this.flush();
    }
  }

  async flush(): Promise<void> {
    if (this.entries.length === 0 || this.flushing) return;
    this.flushing = true;
    const batch = this.entries.splice(0);
    try {
      await this.flushFn(batch);
    } finally {
      this.flushing = false;
    }
  }
```

```javascript
// Citation rendering — DOM-only, no innerHTML.

const CITATION_PATTERN = /\[drawer_([a-z0-9]+)\]/g;

function buildPopover(drawerId, meta) {
  const popover = document.createElement("span");
  popover.className = "citation-popover";
  popover.setAttribute("role", "tooltip");

  const title = document.createElement("strong");
  title.textContent = meta ? `${meta.wing} · ${meta.room}` : drawerId;
  popover.appendChild(title);

  if (meta?.created_at) {
    const date = document.createElement("span");
    date.className = "citation-date";
    date.textContent = new Date(meta.created_at).toLocaleDateString();
    popover.appendChild(date);
  }

  const link = document.createElement("a");
  link.className = "citation-link";
  // VIZ_BASE_URL is set on <body data-viz-base-url="..."> at SSR/static time.
  // Empty string = local /api/familiar/memory route (v0.2 default).
  // Non-empty = mempalace-viz origin (v0.2.1+ when deployed). One config swap.
  const vizBase = document.body.getAttribute("data-viz-base-url") ?? "";
  link.href = vizBase
    ? `${vizBase}/drawer/${drawerId}`
    : `/api/familiar/memory/${drawerId}`;
  link.target = "_blank";
  link.rel = "noopener noreferrer";
  link.textContent = vizBase ? "View in palace ↗" : "View in palace →";
  popover.appendChild(link);

  return popover;
}

function buildCitationButton(rawId, meta) {
  const drawerId = "drawer_" + rawId;
  const wrapper = document.createElement("span");
  wrapper.className = "citation";
  wrapper.setAttribute("data-drawer-id", drawerId);

  const btn = document.createElement("button");
  btn.className = "citation-trigger";
  btn.setAttribute("aria-label", `Palace source: ${drawerId}`);
  btn.textContent = `[${rawId.slice(0, 6)}]`;

  const popover = buildPopover(drawerId, meta);
  wrapper.appendChild(btn);
  wrapper.appendChild(popover);

  btn.addEventListener("click", (e) => {
    e.stopPropagation();
    const isOpen = wrapper.classList.toggle("citation--open");
    if (isOpen && !meta) {
      fetch(`/api/familiar/memory/${drawerId}`)
        .then((r) => r.ok ? r.json() : null)
        .then((data) => {
          if (data) {
            const fresh = buildPopover(drawerId, data);
            wrapper.replaceChild(fresh, wrapper.querySelector(".citation-popover"));
          }
        })
        .catch(() => {});
    }
  });

  return wrapper;
}

/**
 * Render text containing [drawer_xxx] markers into a container.
 * Uses DOM text nodes and elements — never innerHTML.
 */
export function renderWithCitations(container, text, knownMeta = {}) {
  // Clear existing children
  while (container.firstChild) container.removeChild(container.firstChild);

  let lastIndex = 0;
  CITATION_PATTERN.lastIndex = 0;
  let match;

  while ((match = CITATION_PATTERN.exec(text)) !== null) {
    if (match.index > lastIndex) {
      container.appendChild(document.createTextNode(text.slice(lastIndex, match.index)));
    }
    const rawId = match[1];
    const meta = knownMeta[`drawer_${rawId}`] ?? null;
    container.appendChild(buildCitationButton(rawId, meta));
    lastIndex = match.index + match[0].length;
  }

  if (lastIndex < text.length) {
    container.appendChild(document.createTextNode(text.slice(lastIndex)));
  }
}
```

### What this v0.2 ships, framed against the synthesis

- **Phase 0.2** closes the v0.1 gap that left chat polluted by checkpoint noise (`kind=content`).

- **Phase 8** makes citations *visible*, with a one-config-swap path to mempalace-viz once it's deployed.

Each phase is a v∞-seed at minimum cost. None require committing to GraphPalace, rlm, hermes, or the active-inference paradigm. v0.3 can adopt any of them by extending the seams that v0.2 carves.
